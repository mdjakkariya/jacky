## [0.2.0] ‚ÄĒ 2026-06-23

_First version with the changelog; entries before this were curated by hand. From
0.3.0 on, this file is generated automatically from Conventional Commits._

### ūüöÄ Features

- Chat mode: a typed, right-docked drawer alongside voice, with inline tool
  confirmations and easy voice‚áĄchat switching.
- Global hotkeys to drive Jack from any app ‚ÄĒ ‚Ćė‚ĆÉJ summon/dismiss, plus ‚Ćė‚ĆÉC chat,
  ‚Ćė‚ĆÉS settings, ‚Ćė‚ĆÉR report, ‚Ćė‚ĆÉL lock, ‚Ćė‚ĆÉQ quit (shown right-aligned in the menu).
- Full-duplex barge-in: talk over Jack to interrupt, using macOS echo cancellation.
- App & web control (open/close/focus/hide/quit/uninstall apps, open/close a site)
  and an in-app "Report an issue" flow that prefills a GitHub issue.
- Central macOS permission handling: Jack checks Microphone/Automation/Accessibility
  and points you to the right Settings pane when something's missing.

### ūüźõ Bug Fixes

- Voice and chat could run at once (orb talking while you typed); typing now forces
  chat mode and the orb stays hidden during chat.
- Confirmation cards no longer double up on the orb during a chat turn.
- empty_trash reports "already empty" instead of a cryptic ‚ąí128 error, and avoids
  Finder's redundant warning dialog.
- A timed-out confirmation now says it cancelled for lack of a response, not that
  you declined.
- Chat input no longer gets stuck loading: one turn at a time, input locked while
  in flight.

### ‚ö° Performance & Stability

- Voice and chat turns are serialized so they can't corrupt the shared state machine.
- Toggling voice‚áĄchat no longer reconnects the cloud model or reloads speech-to-text.

### ūüõ†ÔłŹ Improvements

- Friendlier, action-specific spoken acknowledgements ("Opening Spotify"); dismiss
  stays silent. Plain-language confirmation prompts that name the target.
- Chat drawer can be moved and resized; shortcut hint shown in the input placeholder.
